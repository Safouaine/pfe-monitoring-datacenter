## Project Overview

Predictive monitoring platform for a Nouvameq datacenter. Collects SNMP sensor data, stores it in InfluxDB, runs a hybrid TimesFM + Random Forest AI pipeline for fault prediction, and serves results via a FastAPI web interface and Grafana dashboards.

## Starting the Full Stack

```bash
docker compose up --build
```

Individual services can be rebuilt and restarted:
```bash
docker compose up --build <service-name>
# e.g.: docker compose up --build ai-engine
```

Service URLs after startup:
- Web platform: http://localhost:8000
- Grafana: http://localhost:3000
- InfluxDB: http://localhost:8086

## Running Services Locally (without Docker)

**Web platform (FastAPI):**
```bash
cd web-platform
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

**AI pipeline (train RF model — run once before the web platform):**
```bash
cd ai-engine
pip install -r requirements.txt
python run_ml_pipeline.py
```

**SNMP collector:**
```bash
cd snmp-collector
pip install -r requirements.txt
python main.py
```

**Anomaly detection (one-shot, requires live InfluxDB):**
```bash
cd ai-engine
python cerveau-anomalie-detector.py
```

**TimesFM standalone forecast + visualization:**
```bash
cd ai-engine
python predict_timesfm.py
# outputs timesfm_forecast.png
```

## Architecture

```
snmp-collector → InfluxDB → Grafana (dashboards)
                          → ai-engine (Isolation Forest anomaly detection)
                          → web-platform (FastAPI + hybrid AI pipeline)
```

### Key data flow for the web platform's `/api/predict` endpoint

1. `web-platform/ai_predictor.py` — loaded once at startup via `@app.on_event("startup")`:
   - Reads `data/datacenter_tunisia_rf_dataset.csv` (CSV with 11 sensor columns + `target`)
   - Loads `advanced_rf_model.pkl` (trained by `ai-engine/run_ml_pipeline.py`)
   - Downloads `google/timesfm-1.0-200m-pytorch` from HuggingFace on first run
2. **TimesFM** forecasts the next 32 time steps for all 11 base sensors using the last 100 rows as context
3. Feature engineering adds rolling means and rack delta temperatures
4. **Random Forest** computes failure probability across the 32-step horizon
5. Response includes `global_risk_percent`, `status` (NORMAL / ALERTE / CRITIQUE), per-sensor danger levels, and a `risk_timeline` array

### Sensor schema (11 base features)

`temp_ext`, `humidity`, `rack1_h`, `rack1_m`, `rack1_b`, `rack2_h`, `rack2_m`, `rack2_b`, `pwr_consumption`, `fuel_level`, `battery_health`

Engineered features added before RF: `temp_ext_rolling10`, `humidity_rolling10`, `rack1_delta_temp`, `rack2_delta_temp`

### InfluxDB configuration

- Org: `Nouvameq`  
- Bucket: `datacenter_metrics`  
- Measurement: `data_center_sensors` (tag: `sensor_name`, field: `value`)  
- Token: hardcoded in `snmp-collector/main.py` and `ai-engine/cerveau-anomalie-detector.py` — override via `INFLUXDB_TOKEN` env var

### SNMP collector

Uses `pysnmp==4.4.12` (pinned — newer versions break the import). Runs with `network_mode: "host"` in Docker so it can reach the physical AKCP sensor at `192.168.1.100`. Override sensor IP with `IP_AKCP` env var.

### Data files (required at runtime)

- `data/datacenter_tunisia_rf_dataset.csv` — primary training/inference dataset
- `data/datacenter_tunisia_2M.csv` — larger raw dataset
- `advanced_rf_model.pkl` — trained RF model (generated by `ai-engine/run_ml_pipeline.py`; must exist before starting the web platform)
- `data/Talend_datawarehouse/` — Talend ETL project for loading CSVs into the data warehouse
- `data/dashboard-1777720263537.json` — Grafana dashboard export (import via Grafana UI)

### TimesFM model

Uses `google/timesfm-1.0-200m-pytorch` (200M parameter foundation model from HuggingFace). Backend is CPU. First load downloads ~800MB of weights; they are cached by HuggingFace's standard cache mechanism. Parameters: `context_len=128`, `horizon_len=32`, `num_layers=20`, `model_dims=1280`.

### PostgreSQL (optional)

`ai-engine/run_ml_pipeline.py` writes predictions to a local PostgreSQL `datacenter-dw` database (`postgresql://postgres:admin@localhost:5432/datacenter-dw`). This is optional and failures are caught; the model is saved regardless.
